## 02 Bitcoin Mining

Bitcoin miner windows application that can mine with CPU, GPU & Block Erupter. 

<Image
  src="/images/project-bitcoin-mining.jpg"
  alt="Photo"
  width={980}
  height={570}
  priority
  className="next-image"
/>

One of the key challanges was to optimize performance using a **midstate** in SHA256 calculation in proof of work (POW) algorithm.

<Image
  src="/images/project-bitcoin-proof-of-work.png"
  alt="Photo"
  width={742}
  height={350}
  priority
  className="next-image"
/>

Bitcoin POW relies on performing a SHA256 hash on the incoming data. 
The **midstate** is the SHA-256 hashing state after hashing the first chunk (64) bytes of the block header. After you get a block header from the
server, you typically hash the first chunk 1 time, and hash the second chunk 2^32 times going through all the possible nonce values.
Because the nonce is in the second chunk, there is no need to hash the first chunk over and over every time you try a new nonce value.

Bitcoin SHA-256 hashing with MIDSTATE optimization.
